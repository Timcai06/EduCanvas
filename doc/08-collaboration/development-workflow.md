# 开发工作流

- 状态：`accepted`

根目录[协作.md](../../协作.md)是新成员的最短指南，本文件说明团队内部怎样组织任务。

## 一个功能的生命周期

```text
Issue说明问题
→ 明确负责人和验收标准
→ 创建功能分支
→ 编码和本地验证
→ 更新文档
→ 创建Pull Request
→ Review和自动检查
→ Squash合并
→ 验证共享环境
```

## Issue至少写清楚

- 背景；
- 目标；
- 不做什么；
- 验收标准；
- 负责人；
- 相关文档；
- 依赖和风险。

## PR大小

优先小PR。一个PR完成一个可解释目标。数据库、接口、前端和文档如果属于同一功能可以一起提交，但不要顺手混入无关重构。

## Review重点

- 是否符合需求；
- 是否引入安全或隐私问题；
- 是否存在并发、幂等和失败恢复问题；
- 是否影响API、数据或Artifact协议；
- 测试是否覆盖关键风险；
- 文档是否同步。
