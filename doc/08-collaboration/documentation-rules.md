# 文档维护规则

- 状态：`accepted`

## 谁修改代码，谁更新文档

文档不是项目负责人最后补写的报告。实现行为发生变化时，相关PR必须同时更新文档。

## 放在哪里

- 产品行为：`01-product`；
- 跨模块架构：`02-architecture`；
- AI、RAG和模型：`03-ai`；
- 表、事件和数据治理：`04-data`；
- 具体工程规范：`05-engineering`；
- 测试、安全和评测：`06-quality`；
- 部署与监控：`07-operations`；
- 团队协作：`08-collaboration`；
- 重大决定：`09-decisions`；
- 阶段计划：`10-planning`。

## 不要做什么

- 不建立`最终版2.md`、`最新版.md`；
- 不复制同一结论到多个文件；
- 不把聊天记录原样贴入文档；
- 不删除历史ADR；
- 不写无法验证的完成状态；
- 不在文档中放真实密钥或学生数据。

## 文档头部

重要文档至少包含：状态、负责人、最后验证时间。暂时未知的负责人可以写`待认领`。
