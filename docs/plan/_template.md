# 阶段计划标题

- 状态：`draft`
- 负责人：`待认领`
- 最后验证时间：`YYYY-MM-DD`
- 对应路线图阶段：[`../10-planning/roadmap.md`](../10-planning/roadmap.md)

## 目标

用一段话说明本阶段结束后可以被验证的结果。

## 范围

- 本阶段会修改或交付什么。

## 非目标

- 明确本阶段不会提前锁定或实现什么。

## 前置事实与依赖

- 链接canonical文档、ADR和前置计划，不复制其全文。

## 验收条件

1. 可观察、可复现的验收条件。

## 实施步骤

1. 按依赖顺序拆分任务，但不要把候选方案写成已接受设计。

## 预期事实回写

| 稳定事实类型 | 目标文档                                              |
| ------------ | ----------------------------------------------------- |
| 产品行为     | `docs/01-product/...`                                 |
| 架构或接口   | `docs/02-architecture/...`或`docs/05-engineering/...` |
| 重大取舍     | `docs/09-decisions/...`                               |

## 验证证据

| 验收项 | 证据                               | 结果      |
| ------ | ---------------------------------- | --------- |
| 待验证 | 命令、测试报告、截图、PR或部署链接 | `pending` |

不得记录密钥、学生数据或无法复现的口头结论。

## 风险与回退

- 风险、触发条件和回退方式。

## 收尾检查表

- [ ] 验收条件均有证据，失败项已明确转入后续计划；
- [ ] 稳定事实已经回写到canonical文档；
- [ ] 重大决策已经写入ADR；
- [ ] 临时方案和过期过程记录已经删除或压缩；
- [ ] 计划已经移入`completed/`并更新`docs/plan/README.md`。
