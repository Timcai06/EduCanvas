# ADR-0006：可信学习事件与服务端判分边界

- 状态：`accepted`
- 日期：2026-07-14
- 负责人：@Timcai06
- 最后验证：2026-07-14
- 实现规格：[学习事件契约](../04-data/learning-event-contract.md)

## 背景

Canvas会产生播放、暂停、答题、分类和提示等客户端交互。现有协议只限制事件名称，`payload`仍允许任意JSON；同时ADR-0004要求从`state_transition`回放教学状态，ADR-0005要求从答题、提示和误区证据重算掌握度。

如果客户端可以直接声明“答对”“完成”或“掌握”，篡改请求、重复提交和模型生成内容都可能污染学习事实源。客户端发生的操作与服务端确认的教学事实必须分层。

## 决定

1. **Canvas交互事件与领域学习事件分离**：`CanvasInteractionEvent`只描述浏览器发生的操作，是不可信输入；`DomainLearningEvent`由服务端在验证会话、答案和状态机规则后生成，是可回放的业务事实；
2. **每种事件使用独立严格Schema**：事件采用按`type`判别的联合，每种payload只允许已评审字段；影响掌握度的数据不得使用`record<string, unknown>`；
3. **只有可信领域事件进入掌握度计算**：动画播放、暂停等过程事件用于观测，不直接改变掌握度；答题和分类提交必须先由服务端判分，再产生`assessment_graded`；
4. **答案与展示数据分离**：浏览器只接收题目展示数据，正确答案和评分规则保存在服务端；客户端提交选项或分类结果，不能提交`isCorrect`作为事实；
5. **事件支持幂等和确定性回放**：领域事件包含幂等键、服务端记录时间和会话内单调序号；客户端时间只作为发生时间，不作为排序的唯一依据；
6. **领域事件归属教学核心**：客户端事件Schema属于`canvas-protocol`；`state_transition`、`assessment_graded`、`hint_recorded`、`misconception_updated`等领域事件在`teaching-core`实现时落地，数据库只负责持久化。

## 原因

- 防止客户端或模型伪造正确率和掌握度；
- 保证状态机、掌握度和学习报告使用同一事实源；
- 让客户端协议、领域规则和数据库持久化可以独立演进；
- 支持网络重试、离线补传、事件回放与审计。

## 后果

- 交互提交需要经过服务端验证，不能只在浏览器本地更新掌握度；
- 为保持即时反馈，UI可以先显示“已提交”，最终判分结果以服务端响应为准；
- `learning_events`表在真实写入链路接入前，需要增加幂等键、会话序号、服务端记录时间和事件来源；
- 新增会影响掌握度的事件时，必须同步更新事件Schema、重算逻辑、测试和数据文档。

## 验证方式

- 契约测试确认每个事件拒绝未知字段和错误payload；
- 篡改测试确认客户端提交`isCorrect`不能直接形成可信事件；
- 幂等测试确认同一提交重试只产生一条领域事件；
- 回放测试确认按会话序号可重建状态并重算掌握度。
