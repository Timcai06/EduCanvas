# ADR-0021：MCP 高风险意图与恢复边界

- 状态：`accepted`
- 日期：2026-07-22
- 决策人：项目负责人
- 上位决策：[第二代 Hybrid Ports 架构](./0020-第二代HybridPorts架构.md)

## 背景

MCP v1 `callTool` 没有标准的耐久任务、幂等键或副作用查询。L2/L3 工具在等待审批时必须保留参数，但公共 `tool_approval_intents` 与 `operation_continuations` 明确禁止保存参数、Credential 和外部正文。若 Worker 在外部系统已提交后崩溃，盲目重试可能产生双写。

## 决定

本 ADR 是可选高风险能力的安全边界，不是近期产品能力优先级。没有真实 write Adapter、平台资格、可信查询或服务端幂等契约时，不以扩展 MCP、自动 verifier 或 L2/L3 能力作为第二代架构收口条件。

1. L2/L3 MCP 只允许 `write`，统一使用受控 Gateway capability `external.mcp.invoke`，且必须同时具备公共 Approval Intent、MCP Adapter 自有耐久仓储和意图加密密钥；任一缺失即 `disabled/durability`。具体工具身份仍由 Tool Call、可信注册和 `resumeRef` 绑定，不能靠一次批准放开其他工具。
2. Adapter 自有 `mcp_tool_intents` 只保存权威范围、带密钥策略摘要、生命周期和 AES-256-GCM 密文。参数与不透明 Credential Handle 在密文内，AAD 绑定 Operation、Actor、Agent、Tool Call、工具和过期时间；幂等摘要使用从主密钥HKDF派生的HMAC密钥，避免低熵参数留下可离线枚举的裸SHA摘要。Secret 仍只由 Credential Broker 短时解析，不进入密文。
3. 密文最长 256 KiB、审批最长 15 分钟。外呼前把状态原子推进为 `dispatching` 并擦除密文；过期且仍为 `prepared` 的密文由五分钟有界维护任务擦除。
4. MCP 高风险工具不能走前台 `invoke`；批准后只能由既有 PostgreSQL continuation 和 Worker 恢复。恢复仍重新计算 Actor、Agent、Notebook Membership、Approval 与 capability。
5. `tool_effects` 是副作用提交事实。若重领时 Effect 已 `committed`，只补齐 Tool、消息和 Operation；若仍 `intended` 或无法确认，收敛为 `outcome_unknown`，绝不再次调用 MCP。未来只有服务端提供受信幂等/查询契约时才能增加自动对账 Adapter。

## 后果

高风险 MCP 可以在审批与进程重启后恢复，同时公共控制账本不携带敏感参数。代价是部署必须管理独立 32 字节密钥，并接受通用 MCP v1 在不确定提交窗口只能诚实报告未知、不能自动重试。密钥轮换和真实 Bearer Credential Broker 在提供生产实现前保持 fail closed。

## 接受记录

本决策是项目负责人已接受的 ADR-0020“完整执行第二代架构、不得牺牲隐私与单一事实源”的具体化，不扩张其产品范围。实现和回滚必须保留 L0/L1 MCP 与普通聊天可用，不能因高风险配置错误拖垮主链。
