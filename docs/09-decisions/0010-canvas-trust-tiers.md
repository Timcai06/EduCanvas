# ADR-0010：Canvas 分层信任模型

- 状态：`accepted`
- 日期：2026-07-16
- 决策人：项目负责人
- 取代：[ADR-0002](0002-controlled-canvas.md)

## 背景

ADR-0002 以一条全局禁令（"模型生成的任意 HTML、JavaScript 或 GSAP 源码不在主页面直接运行"）同时回应了三个性质不同的问题：

1. **注入安全**：模型代码进主页面会暴露 XSS 与数据泄露面；
2. **未成年人内容质量**：教学内容需要稳定、可评测、可访问；
3. **可判分性**：服务端判分、可信学习事件和回放要求结构化契约。

三年间两点事实发生了变化。其一，行业已经用 sandboxed iframe（无 `allow-same-origin`、无网络、postMessage 桥）安全地执行模型生成代码，Gemini Canvas、Claude Artifacts、ChatGPT Canvas 均为此模式——注入安全不再必须靠禁令解决。其二，白名单 Schema 从未真正解决内容质量问题：Artifact 中的题面、选项、文案本就是模型填充的，Schema 校验的是结构而非语义，劣质内容照样能通过校验被渲染。

禁令的真实代价是产品性的：每种新教学形态必须先手写并注册 React 组件，Renderer Registry 是编译期闭集，"生成式交互体验"退化为给预制模板填参数。这与产品北星（ADR-0009：Gemini 式对话 + 可交互产物）和 P4 Artifact Runtime 直接冲突。

## 候选方案

- **A. 维持 ADR-0002 全局禁令**：安全论证最简单，但新形态开发成本恒高，生成式体验永远受限于预注册模板闭集；
- **B. 允许模型代码进主页面（充分转义/审计后）**：注入面直接暴露给未成年人用户，审计不可能穷尽，不可接受；
- **C. 分层信任模型（本决定）**：按"产物是否进入可信学习事实"划分两级信任，判分链路维持 ADR-0002 机制，探索链路引入浏览器沙箱。

## 决定

1. **Tier 1——判分型可信 Artifact（原样保留 ADR-0002 机制）**：凡是产生可信学习事件、进入掌握度或参与服务端判分的 Artifact（现有 `quiz`、`classification_game` 等），必须走白名单 Zod Schema + 预注册 React Renderer + 服务端判分；公开题面与私有判分键继续物理分离；GSAP 时间轴只由可信组件实现；
2. **Tier 2——沙箱探索型产物**：模型生成的 HTML/CSS/JS 只允许在 `<iframe sandbox="allow-scripts">` 中执行，不授予 `allow-same-origin`、`allow-popups`、`allow-top-navigation`；CSP 禁止一切网络外联；与宿主只能通过 postMessage 白名单桥通信，消息进出均经 Zod 校验；
3. **不变量保留**：主页面永不直接执行模型生成的代码。ADR-0002 的这一核心约束原样继承，只是执行环境从"不存在"变为"沙箱"；
4. **Tier 2 不产生可信学习事实**：沙箱内的任何交互不产生可信学习事件、不进入掌握度、不触发教学状态机转移。类型层与 runtime 双重拒绝（Tier 2 产物没有判分键、事件消费方校验 tier）；
5. **数据模型显式标记信任层级**：Artifact 持久化对象携带 trust tier 字段；Studio 的展示与版本化对两级一视同仁，判分与进度消费方必须校验 tier 后才接受输入；
6. **Tier 2 立即、无差别开放**：沙箱预览对通用 Chat 与 K12 Agent 同步开放，不设年龄策略、主题白名单等产品层前置闸门；唯一边界就是沙箱技术边界本身（无 same-origin、禁网络、不进学习事实）。未成年人内容质量继续由对话层既有安全策略覆盖，不为 Tier 2 增设专门审查链路；若试点暴露真实问题，再以独立 ADR 补充，而不是预先设卡。

## 原因

- 注入安全交给浏览器沙箱原语（origin 隔离 + CSP），比"禁止执行"更符合行业已验证的实践，也可被安全测试证伪；
- 可判分性所需的结构化契约在 Tier 1 完整保留，教学正确性由确定性逻辑约束的主轴（ADR-0004/0005/0006）不受影响；
- 内容质量被诚实地归位为独立的内容策略问题——它在两个 tier 中同样存在，不再假装白名单 Schema 能解决它；
- 解锁"沙箱可交互学习区"与 P4 Artifact Runtime：新教学形态不再要求先发版预注册组件。

## 后果

- 需要实现：沙箱宿主组件、postMessage 白名单桥、CSP 配置、Artifact 协议与 SSE 事件中的 Tier 2 产物类型；
- 攻击面从"零执行"变为"沙箱边界"，安全测试必须覆盖：沙箱逃逸、postMessage 畸形消息、资源耗尽（如 `while(1)`）、弹窗与顶层导航劫持；
- [ADR-0002](0002-controlled-canvas.md) 标记为 `superseded`；`CLAUDE.md` 与 [canvas-and-gsap.md](../02-architecture/canvas-and-gsap.md) 同步更新；
- 教学判分路径零变化，现有 Tier 1 测试与实现不动。

## 开放问题

- 沙箱内静态资源（图片、字体）的供给方式：内联 data URI、受控代理还是禁止；
- Tier 2 产物的离线回放与版本 diff 如何呈现；
- Tier 2 → Tier 1 的升级通道是否存在（探索型产物被教师审核后固化为可判分模板）。

## 验证方式

- 安全测试：沙箱内脚本无法读取宿主 cookie/localStorage/DOM，无法发起网络请求，畸形 postMessage 被拒绝并记录；
- 边界测试：Tier 2 产物无法写入学习事件与掌握度（类型层编译失败 + runtime 拒绝各覆盖一次）；
- E2E：一个 Tier 2 产物完成"生成 → 沙箱渲染 → 交互 → 版本化"回路，且全程不产生教学状态变化。
