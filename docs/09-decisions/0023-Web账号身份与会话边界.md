# ADR-0023：Web 账号身份与会话边界

- 状态：`accepted`
- 日期：2026-07-24
- 决策者：项目负责人

## 背景

EduCanvas 需要昵称、头像、跨浏览器登录和密码修改，但现有产品同时存在三种主体来源：
本地 Web/TUI 共用的 `local:owner`、Web 匿名兼容主体，以及注册账号。若注册 Cookie
在 local 部署中静默覆盖 `local:owner`，同一 Notebook 会被拆成两份；若把首版用户名密码
直接称为正式 IdP，又会掩盖找回、MFA、跨客户端建联和生产级滥用防护尚未完成的事实。

## 决策

### 一、主体选择保持显式

- local 部署的 Agent、Notebook、会话和资产始终使用配置的本地主体，默认
  `local:owner`，从而与 TUI/Gateway 保持同一事实归属；
- 非 local 部署中，有效 Web session 可以恢复 `user:<uuid>` 注册主体；没有有效
  session 时才回退匿名兼容主体；
- Web 账号资料在 local 部署中只提供账号界面与凭据走查，不得静默迁移或重新归属
  `local:owner` 数据；
- 匿名历史默认不自动导入注册账号。后续导入必须有显式确认、来源证明和可审计事务；
- TUI/Gateway 使用注册账号前仍需独立完成 session exchange，不能复用浏览器 Cookie。

### 二、密码与 session 是一个事务边界

- 密码使用 Node.js `scrypt`；数据库保存算法版本、受限参数、随机盐和派生值；
- 验证时只接受服务端 allowlist 中的算法版本和参数，不能让数据库内容扩大 CPU/内存成本；
- 密码输入限制为 8–128 个字符；浏览器风险等级只用于提示，服务端独立校验；
- 修改密码必须在同一 PostgreSQL 事务中更新派生材料、撤销全部旧 session、创建当前
  浏览器的新 session；Cookie 只在事务提交后写入；
- session 原始 token 只存在于 HttpOnly Cookie，数据库只保存 SHA-256；30 天是当前
  绝对有效期，不把尚未实现的 `lastSeenAt` 宣称为滑动过期。

### 三、首版账号不是完整生产 IdP

- 用户名注册、登录和密码修改必须有有界 JSON 输入、同源写保护和稳定限流错误；
- 仅进程内限流只能用于 local/单进程开发。staging/production 未接共享限流适配器时必须
  fail-closed，不能伪装为生产保护；
- 头像请求在 multipart 解析前执行总请求硬上限，类型由服务端 magic bytes 判断；
- 对象写入后数据库失败必须补偿删除；替换成功后清理旧对象，失败只记录脱敏错误；
- 邮箱验证、账号找回、MFA、共享限流、审计事件、TUI/Gateway 注册身份建联完成前，
  该适配器不满足生产身份门禁。

## 结果

这条纵切可以真实验证注册、登录、资料、头像、密码轮换和历史归档，同时不破坏本地
Web/TUI 共用 Notebook 的既有契约。代价是 local 账号资料与 Agent 数据归属暂不合并，
生产部署也必须继续保持身份门禁，直到共享认证控制面完成。

## 被否决的方案

- 在 `readAnonymousIdentity()` 中无条件让注册 Cookie 覆盖 local 主体；
- 用进程内 `Map` 冒充多实例生产限流；
- 先更新密码，再分别撤销和新建 session；
- 自动把匿名或 `local:owner` 历史转移给新账号；
- 为首版账号引入第二数据库、Redis 或新的 Agent Runtime。

## 验证方式

- local 模式存在注册 Cookie 时仍解析为 `local:owner`；
- 重复用户名并发注册稳定返回冲突；
- 密码轮换故障回滚后旧密码和旧 session 仍一致，新事务提交后旧 session 全失效；
- 超限 multipart 在 `formData()` 前中止，头像数据库失败不留下新对象；
- 删除非当前历史不改写当前游标，畸形 UUID 与跨主体删除在数据库边界前后稳定拒绝。
