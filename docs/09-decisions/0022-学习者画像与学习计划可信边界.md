# ADR-0022：学习者画像与学习计划可信边界

- 状态：`accepted`
- 日期：2026-07-24
- 决策者：项目负责人

## 背景

EduCanvas 需要根据学生所处学段、学习目标和已有理解调整教学，但“个性化”不能变成模型根据语言、图片或行为猜测年龄、性格、心理状态和能力。短诊断又会影响教学起点与掌握度，因此不能让浏览器或模型自报分数。

P1 还需要把一段普通对话提升为可解释的学习计划：学生知道自己要学什么、哪些是优势、哪些应优先学习，系统也能把这些结构化事实交给统一 Agent Runtime。

## 决策

### 一、画像只保存最小显式声明

- 年龄只保存 `under_13 / 13_to_15 / 16_to_17 / adult / unknown`，不保存出生日期；
- 年级只保存 `primary_school / middle_school / high_school`；
- 来源只接受 `self_declared / guardian_declared / school_asserted`；
- 教学偏好只包含讲解顺序、回答深度、引导方式、首选模态和反馈风格的闭集选项；
- 不保存人格类型、心理标签、生物特征、模型推断年龄或自由文本画像；
- `unknown` 与全部非成年年龄段都采用未成年人安全策略；
- 偏好只能改变表达，不能扩大工具、数据或安全权限。

### 二、学习计划属于 Notebook

- 一个 `learning_goal` 绑定一个 Notebook、学生和冻结的受信课程版本；
- 同一 Notebook 同时最多一个活动 Goal；
- 每个课程版本包含 6–12 个有稳定 ID 的目标，先修只能指向同一图中的前序节点；
- Goal/Objectives 是结构化教育对象，不是 Personal Memory 或 Notebook Memory；
- 普通问答不会自动课程化，只有学生显式建立学习目标时才创建。
- Session 与 Goal 分两次事务建立；Goal 失败时只能补偿删除本次新建且仍无任何 Goal 的 Notebook，恢复缺少 Session→Goal 绑定时必须 fail-closed。

### 三、诊断题面、答案和事实分层

- 浏览器只接收 `PublicDiagnostic`：题面与选项，不含正确答案和内部目标映射；
- 完整课程定义及答案键只存在于受信服务端课程目录；
- 浏览器只提交 `attemptId + questionId + selectedOptionId`，不能提交正确率、目标状态或 mastery；
- 服务端以同一课程版本确定性判分；数据库仓储再次从选择推导结果，不信任调用方提供的摘要；
- Attempt 通过客户端 UUID 和答案 SHA-256 指纹实现安全重放；同 ID 不同答案必须拒绝；
- Attempt、逐题事实、`assessment_graded` 事件、mastery 投影和 `DIAGNOSE→EXPLAIN` 状态事实在同一数据库事务提交；
- 归属不匹配统一返回不可见错误，不能探测其他学生的 Goal、Session 或诊断。

## 结果

- Agent 可以使用可验证、可修改的学段与教学偏好，但不能声称推断了学生性格；
- 学习起点具有确定性证据，模型文字不能成为分数；
- 诊断结果可投影为 `strength / focus / not_started`，并为后续目标推荐提供稳定输入；
- 课程内容目前是代码内受信版本。引入教师课程编辑、模型生成目标图或外部题库前，必须另行设计审核、版本发布与答案保密边界。

## 被否决的方案

- 从聊天措辞、头像、声音或摄像头推断年龄与性格；
- 把自由文本“学习者画像”直接放入 Prompt 和数据库；
- 让模型生成诊断后同时给出并写入分数；
- 把 Goal/Objectives 当作长期记忆或允许同一 Notebook 多个活动计划并行；
- 为这条纵切引入新的 Agent Loop、LangGraph checkpoint 或第二事实源。
