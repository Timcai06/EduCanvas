# ADR-0018：能力授权、Artifact 信任与学习证据

- 状态：`accepted`
- 日期：2026-07-19
- 决策人：项目负责人

## 背景

个人 Agent 通过渠道和设备远程执行任务后，风险不再局限于回答内容。工具可能访问云端账号、文件、设备或外部系统；Canvas 和模型也可能产生看似正确但未经验证的结果。教育平台尤其不能让模型或客户端自行宣布成绩与掌握度。

## 决定

1. 工具有效权限是 Profile allowlist、主体角色、Notebook 所有权、Gateway/Channel 能力、Node 能力、环境策略和安全策略的交集。
2. 能力按风险分层：L0 内容与检索、L1 受控云端连接器、L2 设备能力、L3 主机执行。当前学生默认只允许 L0 和经审核的教育类 L1；L2/L3 不进入学生默认能力。
3. 高风险或有外部副作用的动作必须支持显式审批、幂等、超时、结果未知和审计，不能在渠道消息中用模糊自然语言绕过确认。
4. Canvas/Artifact 分为可信结构化产物与隔离探索产物：影响判分的产物使用严格 Schema、预注册 Renderer、私有判分载荷和服务端验证；模型代码只允许在无同源、禁网络的沙箱中运行，且不得写入学习事实。
5. 浏览器、渠道、Node 和模型产生的事件都是候选输入；只有可信领域服务验证后生成的学习事件可以更新掌握度、误区、课程状态或报告。
6. 结构化课程、掌握度和误区模型属于教育领域服务。算法参数必须版本化、可回放并接受真实评测校准，不能因历史默认值而永久冻结。
7. 未成年人安全是跨 Profile 的不可绕过基线；切换到研究、创作或其他 Profile 不能获得学生身份原本没有的能力。

## 尚未决定

- 成年个人用户是否以及何时开放 L2/L3；
- 第三方 Skill/Plugin 是否允许自行声明工具和渠道；
- 沙箱探索产物面向未成年人的额外内容审查和资源配额。

## 后果

- “工具已注册”不等于“本轮可见”；
- 五维Tool Policy必须由服务端可信来源分别解析；入口capability manifest只描述传输与渲染能力，至多收窄Channel，不能复制成五维grant或增加工具；
- 教师、家长和学生使用同一 Gateway 时仍必须经过角色与资源授权；
- 旧的五阶段状态机和掌握度公式可以继续服务结构化课程，但不再是平台级架构；
- 任何新的设备或渠道能力都必须先定义风险等级、审批和审计。

## 验证方式

- 越权、跨 Notebook、跨角色和渠道能力不匹配均被拒绝；
- 写工具结果未知时不自动重试；
- 客户端提交 `isCorrect` 或模型声称“已掌握”不能形成可信事件；
- 沙箱无法访问宿主 DOM、Cookie、网络或学习事件写入口；
- 权限与学习事件回放可以解释最终决定。

## 实施证据（2026-07-23）

Gateway General、Web General与Web Teaching均通过共享`resolveToolPolicy`解析五维交集。Web Teaching直接消费Gateway权威Route中的Profile与Membership，教学状态白名单只收窄Profile维；浏览器Envelope仅协商输入、输出和Artifact能力，不再携带或授予Teaching Tool capability。纯策略、Profile、可信Route、实际Adapter上界与源码组合边界测试共同覆盖未知来源fail closed、viewer拒绝、恶意manifest不增权和审批结果重新落入最终交集。
