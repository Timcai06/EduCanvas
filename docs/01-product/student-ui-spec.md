# 学生端核心 UI/UX 规格

- 状态：`accepted`
- 相关文档：[产品定义](./product-definition.md)、[Canvas与GSAP](../02-architecture/canvas-and-gsap.md)、[智能体编排](../03-ai/agent-orchestration.md)

## 定位判断标准

打开 EduCanvas 时，学生首先感受到的是「一位正在带我学习的 AI 老师」：对话是课堂，文件和图片是老师使用的教材，Slide、视频和互动实验是老师按需生成的教学产物。Canvas 重要，但不比对话更像产品本体。

交互结构学习 Gemini（对话中心、输入栏「+」统一入口、Canvas 按需分屏）与 NotebookLM（Sources/Chat/Studio 职责三分、回答带来源引用、产物卡片），不复制其品牌视觉。

## 五类对象的层级

| 对象 | 角色 | UI 归宿 | 常驻 |
| --- | --- | --- | --- |
| Chat | 核心交互、导航入口、教学控制面 | 页面主体 + 底部输入栏 | **是（唯一常驻）** |
| Assets 知识资产 | 输入与依据 | 输入框上方可移除标签 + 右侧抽屉；回答中的来源标签 | 否 |
| Canvas | 正在进行的教学互动 | 右侧协作面板，约 40:60 可拖拽 / 全屏 / 收起 | 否，按需 |
| Studio 本课产物 | 教学产物集合 | 顶栏「本课产物」抽屉 + 消息流内产物卡片 | 否 |
| Progress 进度 | 服务端可信状态的轻量映照 | 顶栏阶段徽章 + 掌握度微环，点击展开抽屉 | 徽章常驻，面板不常驻 |

## 布局状态机（纯 UI 状态，与教学脊柱状态机无关）

- **S1 Chat-only（默认）**：对话列居中（约 42rem），输入栏吸底；顶栏只有课程、阶段徽章、掌握度微环、产物入口；
- **S2 Chat + Canvas**：Canvas 右侧展开，默认 40:60，中缝可拖拽（对话 28%–62%）、支持全屏与收起；打开途径只有三种——AI 建议卡、产物卡片、「+」菜单创建；
- **抽屉层**：Assets / Studio / Progress 共用右侧 Sheet（窄屏为底部 Sheet），同屏最多一个，互斥由 `LearnWorkspace` 保证；
- **移动端（<lg）**：默认完整 Chat；Canvas 为全屏页，关闭时恢复原对话滚动位置；禁止压缩桌面分栏。

状态转换动效统一 GSAP（`useGSAP()` 独立 scope、只用 transform/opacity、尊重 `prefers-reduced-motion`、SSR 不执行）。

## 输入栏（Composer）

自上而下四层：上下文标签行（已启用资产，可移除）→ 输入行（「+」｜多行文本框｜语音占位｜发送）→ 状态行（批改中/输入中提示，`aria-live`）。

- Enter 发送、Shift+Enter 换行；文本框自增高至约 6 行后内部滚动；
- 「+」菜单固定两组八项：添加学习材料（上传文件/上传图片/选择课程资料/添加链接）与请老师创建（创建互动演示/生成 Slide/生成测验/更多教学工具）；键盘 ↑↓/Enter/Esc 可达；
- 「请老师创建」类动作不得静默生成大产物；阶段一唯一产物是预置分类游戏，直接以建议卡确认，后续接入参数确认卡；
- 未建设能力一律用教学语言告知（「我还在准备中」），不出现技术错误或伪装成功。

## AI 老师对话体验

- 老师消息不用气泡（头像 + 正文直接落在页面上），学生消息是页面唯一气泡；
- 老师主动提问、一次推进一个学习目标；回答可附来源标签；
- 建议打开 Canvas 前先在对话中征询（〔打开互动演示〕〔继续文字讲解〕）；
- 生成的产物在消息流留下可重新打开的产物卡片；
- 判分结果的数字一律来自服务端反馈 DTO，前端与脚本不得自行判断对错；
- 界面零技术术语（不出现 Artifact、Schema、受控组件等字样）。

## 视觉方向「晨雾」（方向 B 定稿）

- token 定义在 `apps/web/app/globals.css` 的 `@theme`：近白底 `--color-canvas`、蓝灰面 `--color-surface`、唯一强调色靛蓝 `--color-accent`，语义色 good/warn/bad 独立于强调色；
- 品牌字标与 Canvas 标题用衬线 `--font-display`（宋体系），正文系统无衬线；
- 无边框化：消息直接落底、输入栏为灰面胶囊、Canvas 为白色浮卡（圆角 24px + 细线 + 浮影）；
- 组件必须通过 token 取色，禁止散写色值。

## 阶段一实现边界

- 对话由 `features/chat/demo-teacher-script.ts` 的确定性脚本驱动；消息结构与轮次编排器的公开产出对齐，接入真实 Agent 时只替换消息来源；
- 知识资产为演示数据，上传/解析链路未建设；Studio 只聚合当前预置 Artifact；
- 判分、掌握度、进度全部走既有 Server Action 可信链路，客户端不自算；
- 不修改 `packages/*`、数据库结构与可信事件架构。

## 验收标准

1. `/learn` 默认 Chat-only：无侧栏、输入栏可用；
2. Canvas 仅经建议卡/产物卡/「+」打开；可拖拽调宽、全屏、收起；
3. 「+」菜单八项齐全且全键盘可达；
4. 已启用资产显示为输入框上方标签；老师回答可带来源标签；
5. 产物抽屉可重新打开产物；进度徽章常驻、详情在抽屉；
6. 既有判分闭环回归：提交 → 服务端判分 → 反馈 → 进度更新（Playwright e2e）；
7. 移动端默认完整 Chat，Canvas 全屏打开、关闭恢复上下文；
8. 对比度 ≥ 4.5:1、焦点可见、`prefers-reduced-motion` 降级、抽屉 dialog 语义完整。

## 开放问题

- Canvas 宽度是否需要跨会话记忆（当前仅会话内）；
- 语音输入的技术选型与开放时机；
- 参数确认卡的最终交互（依赖第二种可生成产物落地）。
