# 产品定义

- 状态：`draft`

## 产品本体

EduCanvas 是一个以对话为核心的全模态 AI。用户可以在同一条持续会话中提供文本、文件、图片、音频、视频和网页上下文，调用受控 Agent 能力，并生成文档、图片、Slide、视频和可交互 Canvas 等可持久化产物。

## 核心价值

产品不是把多个生成入口拼在一起的工具集合。Chat 负责表达意图和保持上下文；Assets 负责长期知识与媒体资产；Agent Runtime 负责模型、工具和任务编排；Artifact Runtime 与 Studio 负责输出、版本、交互和复用。

## 主要形态

Chat-first：对话是唯一常驻主体和所有能力的触发入口。Canvas 由 AI 建议或用户请求按需展开；文件与媒体可以只服务本轮，也可以显式升级为长期 Asset；生成产物进入 Studio，而不是混入附件列表。专业 Agent 通过能力注册接入同一体验，不另建一套产品外壳。

## 平台能力层级

1. **Chat**：通用对话、流式响应、取消、恢复和上下文连续性；
2. **Assets**：多模态附件、长期资产、版本、解析状态和来源；
3. **Agent Runtime**：模型路由、工具授权、任务运行、Trace和安全策略；
4. **Artifacts**：受控生成的文档、图片、Slide、视频和交互内容；
5. **Vertical Agents**：K12教学、研究、创作等专业能力包。

## 首个垂直产品

K12 AI 教师是首个专业 Agent，用于验证多模态资产、受控工具、Canvas Artifact和可信领域事件能否形成完整闭环。其课程、状态机、掌握度、误区、服务端判分和学习进度属于教学领域，不得成为通用 Agent Runtime 的必选概念。

K12 用户包括学生、教师和内容管理员；具体竞赛目标见[K12垂直产品简报](../00-overview/project-brief.md)，学生端当前规格见[学生端核心UI/UX规格](./student-ui-spec.md)。

## 非目标

- 不把 EduCanvas 等同于单一 AI 教师或单一课程应用；
- 不为每个垂直 Agent 复制 Chat、资产、模型网关和 Artifact 基础设施；
- 第一阶段不覆盖所有K12教材；
- 不以长视频生成作为核心交互；
- 不让大模型自由生成并执行任意网页代码；
- 不依赖单一模型供应商；
- 不以虚拟人外观代替教学效果。
