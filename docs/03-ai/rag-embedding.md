## 检索流程

```text
问题分类
→ 学段/教材/知识点过滤
→ 查询改写
→ PostgreSQL全文召回
→ pgvector向量召回
→ RRF融合
→ Reranker
→ 去重与来源权威校准
→ 返回证据、页码和置信度
```

## 向量空间必须记录

- 供应商；
- 模型ID和固定Revision；
- 维度；
- 模态；
- 距离算法；
- 是否归一化；
- Query/Document Instruction；
- Instruction Hash；
- 切块器版本；
- 创建时间和启用状态。

不同模型即使维度相同，也不能混在一个向量空间。

## 教材切块

- 保留教材、年级、章节和知识点；
- 定义、示例、练习和误区分别标记；
- 采用父子块而不是只按固定Token切割；
- 图片保存OCR、描述、页码和多模态向量；
- 公式、代码和表格不能被无意义截断。

## 模型迁移

创建新空间、双写、后台回填、离线评测、线上Shadow、灰度切换、保留回滚窗口，最后再删除旧索引。

## 评测指标

- Recall@10/20；
- MRR@10；
- nDCG@10；
- 错学段召回率；
- 权威来源命中率；
- p50/p95延迟；
- 单次成本；
- 最终答案引用覆盖率。
