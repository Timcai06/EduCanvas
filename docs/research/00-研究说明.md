# 架构研究说明

- 状态：`accepted`
- 负责人：项目负责人
- 最后验证时间：2026-07-21

`docs/research/` 保存仍在验证期的源码研究、能力盘点、对照实验设计和研究结论。它帮助形成 ADR，但不是产品、架构或实现事实的 canonical 来源。

## 一、与正式文档的关系

```text
源码与官方资料
  -> research：事实、推断、实验、反例
  -> proposed 架构或 ADR：候选方案与决策门
  -> accepted ADR：负责人确认的重大决策
  -> canonical 文档：当前实现和稳定边界
  -> plan：按已接受决策执行
```

- 当前实现以 `docs/01-product` 至 `docs/07-operations`、源码和测试为准；
- 重大技术选择只有进入 `docs/09-decisions` 且状态为 `accepted` 后才生效；
- `proposed` 文档可以指导实验，不能授权生产迁移；
- active plan 记录研究任务和验收，不复制完整研究正文；
- 快照注明适用提交或核验日期，后续不悄悄改写成新事实。

## 二、证据等级

| 标记 | 含义                                             | 可否直接形成决策         |
| ---- | ------------------------------------------------ | ------------------------ |
| 事实 | 当前仓库源码、测试、官方源码或官方文档可直接证明 | 仍需结合 EduCanvas 约束  |
| 推断 | 多个事实支持的架构判断                           | 需要反例与负责人确认     |
| 实验 | 可复现 fixture 或故障注入结果                    | 需要记录版本、命令和边界 |
| 假设 | 尚未通过对照实验的候选方向                       | 不可                     |

外部项目优先使用官方仓库、官方文档、许可证和发布记录。第三方反编译、泄漏或不完整源码只能提供模式线索，不能作为版权来源、接口保证或 ADR 的单一依据。
